# Billing and Payments

TinkerBunker uses a seat-based licensing model for course access.

---

## How Billing Works

| Concept | Description |
|---|---|
| **Seat** | A license for one student to access one course |
| **Pricing** | Set per course by Admin Publishers or Super Admins |
| **Coupons** | Discount codes created by Sales users |
| **Payment** | Processed through Razorpay integration |

---

## Who Manages What

| Area | Role |
|---|---|
| Set course pricing | Admin Publisher, Super Admin |
| Create coupons | Sales |
| Purchase seats | Partner, Institute |
| Process payments | Razorpay (automated) |

---

## Payment Flow

1. Partner or Institute selects a course and number of seats.
2. Applies a coupon if available.
3. Completes payment through Razorpay.
4. Seats are allocated immediately after payment.
