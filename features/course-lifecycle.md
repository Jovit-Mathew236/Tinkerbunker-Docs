# Course Lifecycle

Courses on TinkerBunker follow a structured pipeline from creation to delivery.

---

## Statuses

| Status | Description |
|---|---|
| **Draft** | Course is being built by an Admin Creator |
| **Pending Review** | Submitted for review by an Admin Publisher |
| **Changes Requested** | Sent back to the Creator with feedback |
| **Approved** | Reviewed and ready for publishing |
| **Published** | Live and available for enrollment |
| **Archived** | Removed from active use |

---

## Who Does What

| Step | Role |
|---|---|
| Create content | Admin Creator |
| Submit for review | Admin Creator |
| Review and approve | Admin Publisher |
| Publish | Admin Publisher |
| Assign to partners | Sales |
| Override any status | Super Admin |

---

## Status Transitions

- **Draft** can move to **Pending Review** (by the Creator).
- **Pending Review** can move to **Approved** or **Changes Requested** (by the Publisher).
- **Changes Requested** returns to the Creator, who can edit and resubmit.
- **Approved** can move to **Published** (by the Publisher).
- **Super Admin** can change any status at any time.
