# Course Lifecycle

---

Every course on TinkerBunker goes through a defined workflow before students can access it.

---

## Stages

| Stage | What Happens | Who's Involved |
|---|---|---|
| **Draft** | Course is being created with chapters, pages, and tests | Content Creator |
| **Pending Review** | Course is submitted for review | Content Creator, Publisher |
| **Approved** | Course passes review and is ready to go live | Publisher |
| **Rejected** | Course needs changes before it can be published | Publisher, Content Creator |
| **Live** | Course is published and available to users | Everyone |

---

## How It Flows

Draft is created by the content team. Once ready, it's submitted for review. The publisher either approves it (making it available) or rejects it with feedback. Rejected courses go back to draft for revisions.

{% hint style="info" %}
Only approved courses appear in the course catalog for students and teachers.
{% endhint %}
