# Institute Dashboard

The Institute Dashboard is the central hub for institute administrators. From here, you can manage every aspect of your institute — classrooms, courses, teams, student and teacher requests, analytics, and remote devices.

<figure><img src="../.gitbook/assets/institute-dashboard-overview.png" alt="Institute Dashboard Overview"><figcaption><p>Institute Dashboard — main navigation and summary cards</p></figcaption></figure>

---

## Navigation

The dashboard provides quick access to the following sections:

| Section             | Description                                                        |
| ------------------- | ------------------------------------------------------------------ |
| **Classrooms**      | Create, edit, and manage classrooms. Assign teachers and students. |
| **Courses**         | View and manage courses assigned to your institute.                |
| **Teams**           | Add team members and configure scoped access flags.                |
| **Institute Stats** | View institute-level analytics, performance data, and adoption.    |
| **Join Requests**   | Approve or deny student and teacher join requests.                 |
| **Remote Devices**  | Manage receiver devices used for live quiz sessions.               |

---

## Dashboard Summary Cards

When you first land on the dashboard you will see summary cards providing a snapshot of institute activity:

- **Total Students** — number of enrolled students across all classrooms.
- **Total Teachers** — number of teachers currently assigned.
- **Active Classrooms** — classrooms with at least one linked course and enrolled students.
- **Pending Requests** — student or teacher join requests awaiting approval.

{% hint style="info" %}
Summary card counts refresh each time you load the dashboard. For historical trends, visit the **Institute Stats** section.
{% endhint %}

---

## Quick Actions

From the dashboard you can perform the following actions without navigating away:

1. **Create Classroom** — opens the classroom creation form directly.
2. **View Pending Requests** — jumps to the approval queue.
3. **Invite Team Member** — opens the team invitation flow.

{% hint style="warning" %}
Team members see a filtered version of the dashboard based on their access flags. For example, a member with only `classroom_operation` access will not see the Courses or Teams sections.
{% endhint %}
