# Platform Overview

TinkerBunker is an EdTech platform for managing courses, classrooms, tests, and educational workflows across institutions.

---

## How It Works

Content flows through a structured pipeline:

| Stage | Who | What Happens |
|---|---|---|
| **Creation** | Admin Creator | Builds courses with chapters, pages, and embedded tests |
| **Review** | Admin Publisher | Reviews submitted courses, requests changes or approves |
| **Publishing** | Admin Publisher | Makes approved courses live and available |
| **Distribution** | Sales / Super Admin | Assigns courses to partners and manages pricing |
| **Delivery** | Partners / Institutes | Distributes courses to schools and classrooms |

---

## The Role Hierarchy

TinkerBunker has 11 roles. On the admin side, the hierarchy works like this:

| Level | Role | Reports To |
|---|---|---|
| Top | Super Admin | -- |
| Platform | Admin Creator | Super Admin |
| Platform | Admin Publisher | Super Admin |
| Field | Sales | Admin Publisher |
| Field | Partner | Sales |
| School | Institute | Partner |

Sales invites Partners. Partners invite Institutes. Institutes approve Students and Teachers who self-register.

## Key Admin Capabilities

| Capability | Available To |

| Manage sales team | Admin Publisher |
| Manage partner relationships | Sales |
