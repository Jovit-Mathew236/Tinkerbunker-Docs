# Taking Tests

Tests on TinkerBunker help you assess your understanding of course material. There are several types of tests, each serving a different purpose. This guide covers how to access, navigate, and complete tests.

---

## Types of Tests

| Test Type | Description | Access |
| ------------------- | ----------------------------------------------------------------------------- | ----------------------------------------- |
| **Assigned Tests** | Created and assigned by your teacher. May have a deadline and grading. | Dashboard → Upcoming, or Classroom page. |
| **Practice Tests** | Self-paced tests for revision. No deadlines, no grading by teachers. | Available within courses or test catalog. |
| **View-Only Tests** | Published test results you can review. You cannot re-attempt these. | Accessible from your test history. |

{% hint style="info" %}
Assigned tests may have a time limit and a deadline. Make sure to complete them before the due date to avoid missing marks.
{% endhint %}

---

## Accessing Tests

{% tabs %}
{% tab title="Assigned Tests" %}
1. Navigate to your **Dashboard**.
2. Look under the **Upcoming** section for tests assigned by your teacher.
3. Click on the test to view details (duration, number of questions, deadline).
4. Click **Start Test** to enter the test screen.

Alternatively, you can find assigned tests under your **Classroom** page.
{% endtab %}

{% tab title="Practice Tests" %}
1. Navigate to **Courses** and open a course you are enrolled in.
2. Look for practice tests linked to chapters or available in the course overview.
3. Click on the practice test and then click **Start**.
4. Practice tests can be re-attempted as many times as you like.
{% endtab %}

{% tab title="View-Only Tests" %}
1. Navigate to your test history or the specific assignment.
2. Click on a completed test marked as **View Only**.
3. You can review your answers, correct answers, and explanations (if provided).
4. No modifications or re-attempts are possible.
{% endtab %}
{% endtabs %}

## The Test Screen

Once you start a test, you enter the dedicated **Test Screen** — a focused, distraction-free interface.

![Test Screen](.gitbook/assets/test-screen.png)

### Test Screen Layout

| Element | Description |
| ----------------------- | ---------------------------------------------------------------------------- |
| **Timer** | Displays remaining time (for timed tests). Counts down in real time. |
| **Question Display** | Shows the current question with its options or input fields. |
| **Question Navigation** | A panel (usually on the side or bottom) with numbered buttons for each question. |
| **Status Indicators** | Color-coded indicators showing answered, unanswered, and flagged questions. |
| **Submit Button** | Finalizes and submits your test. |

---

## Question Navigation

The question navigation panel lets you move between questions freely.

| Status | Color/Indicator | Meaning |
| -------------- | --------------- | ------------------------------------------------ |
| **Unanswered** | Grey | You have not selected or entered an answer yet. |
| **Answered** | Green | You have provided an answer for this question. |
| **Flagged** | Orange | You marked this question for review later. |
| **Current** | Highlighted | The question you are currently viewing. |

{% tabs %}
{% tab title="Moving Between Questions" %}
- Click the **Next** button to move to the next question.
- Click the **Previous** button to go back.
- Click any numbered button in the navigation panel to jump directly to that question.
{% endtab %}

{% tab title="Flagging a Question" %}
- Click the **Flag** or **Mark for Review** button on any question.
- The question number in the navigation panel will turn orange.
- Before submitting, review all flagged questions.
{% endtab %}
{% endtabs %}

---

## Timer

{% hint style="warning" %}
For timed tests, once the timer reaches zero the test is **auto-submitted** with whatever answers you have provided up to that point. Keep an eye on the timer and allocate your time wisely.
{% endhint %}

- The timer is visible at the top of the test screen.
- For untimed practice tests, no timer is displayed.

---

## Submitting a Test

1. After answering all questions (or when you are ready), click the **Submit** button.
2. A confirmation dialog will appear showing:
   - Total questions answered
   - Total questions unanswered
   - Total questions flagged for review
3. Confirm submission to finalize your test.

{% hint style="warning" %}
Once submitted, you cannot modify your answers. Double-check your responses before confirming.
{% endhint %}

---

## After Submission

| Scenario | What Happens |
| ---------------------- | ----------------------------------------------------------------------------- |
| **Practice Test** | Results are shown immediately with scores and correct answers. |
| **Assigned Test** | Results may be shown immediately or after teacher evaluation, depending on the test configuration. |
| **View-Only Test** | A read-only view of your submitted answers and the correct answers is available. |

---

## Tips for Taking Tests

{% hint style="info" %}
- Read each question carefully before answering.
- Use the **Flag** feature to mark questions you want to revisit.
- For timed tests, do not spend too long on a single question. Move on and return later if time permits.
- Check the question navigation panel before submitting to ensure no question is left unanswered unintentionally.
{% endhint %}
